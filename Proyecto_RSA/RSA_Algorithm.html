Definiciones (conceptuales y formales)

Definiciones conceptuales

Antes de entrar en materia, es preciso definir unos conceptos importantes:

  • Matemáticas discretas: Es el área de las matemáticas que se encarga del estudio de los conjuntos discretos (es decir, de aquellos conjuntos finitos o contables).
  • Aritmética modular: Define como realizar operaciones sobre congruencias O, en otras palabras, permite el uso de “trucos” para simplificar cálculos. Esto es de gran utilidad cuando se necesiten hacer cálculos de números muy grandes y no se dispone de mucho espacio en memoria. Es decir, la aritmética modular permite ahorrar recursos computacionales.
  • Algoritmo: Conjunto ordenado y finito de operaciones que permite hallar la solución de un problema.
  • Mensaje: Es el objeto de la comunicación. Es la información que el emisor envía al receptor a través de un medio de comunicación.
  • Cifrado: Es el proceso de codificación de la información. Es una técnica que consiste en escribir mensajes en clave, protegiendo el contenido del mismo. Esto ocurre mediante un algoritmo que vuelve incomprensible el mensaje y a la generación de clave que permite comprenderlo.

Definiciones formales

Ya con las definiciones conceptuales planteadas, es necesario brindar las definiciones matemáticas formales que darán forma a la solución que se planteará.

  • Congruencia: Sea n un entero positivo, los enteros a y b son congruentes modulo n si cada uno tiene el mismo residuo en la división por n. Si es así, se escribe: a ≡ b (mod n).
  • Euler Totient: La función ɸ de Euler u Euler Totient es ɸ(n) = |{x: 1 ≤ x ≤ n ∧ mcd(n, x) = 1}|. O sea, el número de enteros positivos x ≤ n que no tienen divisores comunes con n.
  • Máximo común divisor (GCD, en inglés): El máximo común divisor de dos enteros a y b es el mayor npumero entero positivo n que es divisor de a y b.
  • Número primo: Es un número natural mayor que 1 que tiene únicamente dos divisores positivos distintos: él mismo y el 1.
  • Número primo relativo: Dos números son primos relativos (o coprimos) si el máximo común divisor entre ellos es 1.
  • Inverso multiplicativo: Un inverso multiplicativo módulo n de a es el número r tal que a * r ≡ 1 (mod n).
  • Teorema de la división: Suponga que a ∈ Z y n ∈ Z > 0. Entonces, existen enteros q y r tales que: a = q ∗ n + r, 0 ≤ r < n. q es el cociente y r es el residuo.
  • Algoritmo de Euclides: Es usado para encontrar el Máximo Común Divisor y se describe de la siguiente manera. Suponga que a y b son enteros positivos:
    1. Aplicar el teorema de la división repetidamente.
      • a = q1 ∗ b + r1
      • b = q2 ∗ r1 + r2
      • r1 = q3 ∗ r2 + r3
      • r2 = q4 ∗ r3 + r4
    2. Detenerse cuando el residuo sea 0.
    3. El M.C.D. es el residuo de la ecuación anterior a la última (por ejemplo, si r4 = 0, entonces, r3 es el M.C.D.).
  • Identidad de Bézout: Dados dos enteros a y b, ambos diferentes de 0 , y siendo d el Máximo Común Divisor. Entonces, existen enteros v y w tales que: d = av+bw. Existe un algoritmo que permite hallar las constantes v y w:
    1. Aplicar el teorema de la división repetidamente.
      • a = q1 ∗ b + r1
      • b = q2 ∗ r1 + r2
      • r1 = q3 ∗ r2 + r3
      • r2 = q4 ∗ r3 + r4
    2. Detenerse cuando el residuo sea 0.
    3. Aislar todos los residuos (excepto el que es igual a cero) y los números a y b. Es decir, despejarlos de las ecuaciones del teorema del residuo (por ejemplo, r1 se expresaría como r1 = a − q1 ∗ b).
    4. Hacer una “sustitución hacia atrás” (haciendo operaciones de arriba hacia abajo con las ecuaciones despejadas, es decir, desde r1 hasta rn) hasta que se llegue a la ecuación que incluye el penúltimo residuo (el último que es mayor que 0). Esa estará expresada en términos de a y b; las constantes que acompañen esos términos serán v y w.
      • Nota 1: No se deben operar los residuos, solo se deben operar las constantes y/o términos comunes que los acompañen.
      • Nota 2: Cuando a o b (o ambos) son negativos, simplemente se hace la “sustitución hacia atrás” como si los números fueran positivos, luego, se reemplazan (v y w) en la identidad de Bézout conforme su signo.

El algoritmo RSA

Habiendo mostrado las definiciones tanto conceptuales como formales, es necesario describir el algoritmo RSA (la solución que se implementará).

El algoritmo de cifrado RSA es un sistema criptográfico asimétrico ampliamente utilizado para garantizar la seguridad de la comunicación y proteger la información sensible. Fue creado en 1977 por los criptógrafos Ron Rivest, Adi Shamir y Leonard Adleman en el Instituto de Tecnología de Massachusetts (MIT).

La motivación detrás del desarrollo del algoritmo RSA fue proporcionar un método de cifrado seguro que pudiera garantizar la confidencialidad y autenticidad de los datos transmitidos a través de canales inseguros, como Internet. El cifrado RSA se basa en el concepto de clave pública y clave privada, también conocido como criptografía asimétrica.

El problema que resuelve el algoritmo RSA es la necesidad de compartir información confidencial de manera segura sin que los interceptores puedan acceder a ella. El algoritmo permite a las partes comunicarse de forma segura a través de un canal inseguro sin necesidad de compartir previamente una clave secreta.

El algoritmo funciona de la siguiente manera:

  1. Escoger 2 números enteros lo suficientemente grandes. Estos serán p y q.
  2. Calcular n = p * q.
  3. Calcular ɸ(n) = (p - 1)(q - 1).
  4. Escoger un número a con las siguientes características:
    • 2 ≤ a ≤ ɸ(n)
    • gcd(a, ɸ(n)) = 1
    • a y ɸ(n) sean primos relativos (coprimos)
  5. Calcular b = a^(-1) módulo ɸ(n).
  6. Publicar n y a.
  7. Mantener p, q y b ocultos.

Las funciones de encriptación y desencriptación son las siguientes:

  • e(x) = x^a mod n -> Función de encriptación
  • d(y) = y^b mod n -> Función de desencriptación

Ejemplo

El algoritmo RSA puede verse aplicado así:

  1. Generación de claves:
    • Pepito (el emisor) elige dos números primos grandes, p y q. Supongamos que p = 17 y q = 11.
    • Se calcula el producto n = p * q. En este caso, n = 17 * 11 = 187.
    • Se calcula la función phi de Euler de n, ɸ(n) = (p - 1) * (q - 1) = 16 * 10 = 160.
    • Se elige un número a que sea coprimo con ɸ(n) y menor que ɸ(n). Supongamos que a = 7. (7 y 160 son coprimos).
    • Se calcula b, el inverso multiplicativo de a módulo ɸ(n). En este caso, b = 23 (7 * 23 ≡ 1 mod 160).
  2. Intercambio de claves:
    • Pepito (el emisor) hace público n y a (la clave pública).
    • Fulanito (el receptor) quiere enviar un mensaje a Pepito. Primero, convierte el mensaje en un número. Supongamos que el mensaje es "8" y su representación numérica es 8.
    • Fulanito utiliza la clave pública de Pepito (n y a) para cifrar su mensaje. Cifrado: 8^7 mod 187 = 129.
  3. Descifrado del mensaje:
    • Pepito usa su clave privada (b) para descifrar el mensaje cifrado de Fulanito. Descifrado: 129^23 mod 187 = 8.

Paso 3 del algoritmo RSA: calcular Totient de Euler de n.

Tal como se menciono en su definición formal, el Totient de n en el contexto del algoritmo RSA puede ser calculado como (p - 1)(q - 1). La función computeTotientEuler() se encarga de ello.

Paso 4 del algoritmo RSA: escoger un número a aleatorio que cumpla: 2 ≤ a ≤ ɸ(n), gcd(a, ɸ(n)) = 1, a y ɸ(n) sean primos relativos (coprimos).

Las funciones chooseGCDAandTotient(), divisionAlgorithm(A, B), euclidesAlgorithm(A, B) y insertInMatrix(A, Q, B, R) se utilizan para calcular el máximo común divisor entre dos números y verificar si son primos relativos. También se realizan divisiones sucesivas utilizando el algoritmo de Euclides y se almacenan los residuos y las igualdades en matrices para su posterior uso.

La función chooseGCDAandTotient() elige un número a que sea mayor o igual a 2 y menor o igual que totientN, de modo que a y totientN sean primos relativos. Utiliza un bucle while para generar números aleatorios en el rango requerido y utiliza el algoritmo de Euclides (euclidesAlgorithm()) para verificar si el máximo común divisor (MCD) entre a y totientN es igual a 1.

La función divisionAlgorithm(A, B) implementa el algoritmo de la división. Recibe dos números A y B, realiza la división entera entre ellos y calcula el cociente Q y el residuo R. Luego, imprime los resultados utilizando la función printAlgorithm() y guarda los valores en una matriz utilizando la función insertInMatrix(). Finalmente, devuelve los valores B y R.

La función euclidesAlgorithm(A, B) implementa el algoritmo de Euclides para calcular el MCD entre dos números A y B. Utiliza una variable global DicResidues para almacenar los residuos obtenidos durante el proceso y una variable global equalityBase para almacenar una matriz que representa las igualdades sucesivas. La función utiliza la función divisionAlgorithm() para realizar las divisiones sucesivas y determinar el MCD. Luego, imprime el resultado y verifica si el MCD es igual a 1, lo que indica que A y B son primos relativos.

La función insertInMatrix(A, Q, B, R) inserta los valores A, Q, B y R en la matriz DicResidues. Si el residuo R es igual a 1, también asigna los valores correspondientes a la variable global equalityBase.

def euclidesAlgorithm(A, B):
    GCD = 0
    global DicResidues
    global equalityBase
    DicResidues = {}
    equalityBase = []

    print(bold("Algoritmo de la División:") + " a = q•b + r\n")
    print(bold("a = ") + str(A) + ", " + bold("b = " + printYellow(bold("φ(n) = "))) + str(B) + "\n")
    resultA, resultB = divisionAlgorithm(A, B)
    while resultB != 0:
        if resultA % resultB == 0:
            GCD = resultB
        resultA, resultB = divisionAlgorithm(resultA, resultB)

    if resultA ==1:
        print("\n" + bold("El Máximo Común Divisor entre " + str(A) + " y " + str(B) + " es: ") + printYellow(str(resultA)) + ".\n" +
                printYellow("  ---- Por lo tanto, a y φ(n) SI son Primos Relativos ✅ ----  ") + "\n\n\n" +

                 printRed("  ---- Por lo tanto, a y φ(n) NO son Primos Relativos ❌ ----  ") + "\n\n\n")
    return GCD
def insertInMatrix(A, Q, B, R):
    if R>=1:
        DicResidues[R] = [A, Q, B]
        if R==1:
            global equalityBase
            equalityBase =[R, [1, A], [Q, B]]

    global encryptedMsg
    decryptedValues = []
    encryptedMsgDecoded = base64.b64decode(encryptedMsg.encode('utf-8')).decode('utf-8')

Conclusiones

Se pueden concluir las siguientes ventajas respecto de la implementación de este algoritmo:

  1. Seguridad: El algoritmo RSA proporciona un alto nivel de seguridad en la comunicación de datos, ya que se basa en la dificultad de factorizar grandes números primos. La fortaleza del algoritmo radica en la dificultad de calcular los factores primos de un número compuesto lo suficientemente grande.
  2. Uso de cables púbicas y privadas: El algoritmo RSA utiliza un par de claves, una pública y una privada. La clave pública se utiliza para encriptar el mensaje, mientras que la clave privada se utiliza para desencriptar el mensaje. Esto permite una comunicación segura, ya que solo la parte que posee la clave privada puede descifrar los mensajes encriptados. Esto se realiza mediante la generación de claves a partir de números primos aleatorios grandes. La seguridad del algoritmo depende de mantener en secreto la clave privada.
  3. Proceso de encriptación y desencriptación: En el proceso de encriptación, el mensaje se divide en bloques y cada bloque se encripta utilizando la clave pública y una operación exponencial modular. El mensaje encriptado resultante es seguro para su transmisión. Mientras que el receptor del mensaje encriptado utiliza la clave privada para desencriptar el mensaje. Mediante una operación exponencial modular con la clave privada, se obtiene el mensaje original.
  4. Uso de la base 64: En la implementación efectuada, el mensaje encriptado se codifica utilizando base64 antes de su transmisión. Esto permite representar los valores encriptados como una cadena de texto legible que facilita su almacenamiento y transmisión.

La implementación del algoritmo RSA que se realizó es eficiente y confiable para garantizar la seguridad al momento de transmitir datos. No obstante, es preciso considerar que el rendimiento de este algoritmo podría verse afectado por el tamaño de los números primos utilizados y la longitud del mensaje a encriptar. Adicionalmente, se deben proteger las claves privadas apropiadamente, para que la integridad del mensaje no se vea comprometida.
